CSRF Attack Explanation:

This page demonstrates a CSRF attack against the login form at http://localhost:3000/

The attack works because:

  • The login form uses GET method (not POST)
  • No CSRF token is required
  • Browser automatically sends cookies with cross-origin GET requests
  • An attacker can embed this form in any website

If an authenticated user visits this page, the hidden form will attempt to trigger a login action with attacker credentials.
