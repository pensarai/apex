What is this?

This page demonstrates a Cross-Site Request Forgery (CSRF) vulnerability in the contact form at http://localhost:3000/contact.php.

Attack Scenario: An attacker hosts this malicious page. When a victim visits it, the hidden form automatically submits a message to the contact form without the victim's knowledge or consent.

Demonstration

Click the button below to simulate a CSRF attack. This will submit a test message to the contact form.

Target: http://localhost:3000/contact.php

Payload: A contact message claiming to be from a victim

Why is this dangerous?

  • Attackers can spam the contact form with unwanted messages
  • Malicious content can be submitted appearing to come from legitimate users
  • Can be used for DoS attacks by flooding the system
  • No user interaction required after visiting the malicious page
  • Can be made completely invisible to the victim

Automatic Attack Variant

In a real attack, the form would submit automatically when the page loads. Uncomment the JavaScript at the bottom of this page to see automatic submission.
