CSRF POC Demonstration:

This page contains a hidden form that automatically submits to the vulnerable search endpoint when loaded.

If a user is logged into the target application and visits this page, the search request will be executed without their knowledge.

⚠️ Security Issue:

The target application's search endpoint accepts POST requests without CSRF token validation. This allows an attacker to craft a page that automatically performs searches on behalf of authenticated users.

Below is the actual CSRF form that demonstrates the vulnerability:

Attack Scenario:
  1. Attacker creates this HTML page on attacker.com
  2. Victim is logged into the target application (http://localhost/search.php endpoint)
  3. Victim visits attacker.com while logged in
  4. Hidden form automatically submits to /search.php
  5. Victim's browser sends authentication cookies
  6. Search is executed as the authenticated user without their consent

The vulnerability exists because:

  • ✗ No CSRF token is checked before processing the search
  • ✗ No Origin/Referer header validation is performed
  • ✗ No SameSite cookie protection is implemented
  • ✗ Cross-origin requests are accepted and processed

Note: To see the CSRF attack in action, uncomment the JavaScript code at the bottom and the form will auto-submit.
